# 贡献指南

## 简化贡献指南（面向学生开发者）

感谢您对 SplitMind 项目的关注和支持！作为一个学生开发的项目，我们欢迎简单直接的贡献。

## 快速开始

### 1. 环境准备

```bash
# 安装开发依赖
pip install -e ".[dev]"

# 运行测试
pytest

# 格式化代码
black splitmind
```

### 2. 如何贡献

#### 报告问题
- 在 GitHub Issues 中创建新 issue
- 标题要清晰，描述要简单
- 说明问题的具体表现和复现步骤

#### 提交代码
1. **Fork 仓库**
2. **修改代码**
3. **运行测试**
4. **提交 PR**，描述你做了什么

### 3. 我最需要的贡献

1. **Bug 修复**（最高优先级）
2. **简单的功能改进**
3. **文档修复**
4. **测试用例补充**

### 4. 预期

- **响应时间**：我会尽量在周末查看 issues 和 PR
- **反馈**：我会给出简单直接的反馈
- **合并**：如果代码能工作且风格一致，很可能会被合并

### 5. 为什么贡献？

- 学习如何参与开源项目
- 建立你的作品集
- 帮助让 AI 更注重隐私
- 获得 LLM 和多智能体系统的经验

## 行为准则

- 尊重他人，使用友好的语言
- 接受建设性批评
- 对初学者友好，提供帮助

## 感谢

无论贡献大小，都非常感谢！每一个贡献都是对项目的支持。

🚀 让我们一起让 SplitMind 变得更好！
