# 行为准则

## 我们的承诺

为了创建一个开放、包容、友善的社区，SplitMind 项目的所有贡献者、维护者和参与者承诺：

- 无论年龄、体型、残障情况、种族、性别认同与表达、经验水平、国籍、个人外貌、种族、宗教或性取向如何，都将尊重每一位社区成员
- 创造一个没有骚扰的环境，无论线上还是线下
- 以专业和友善的方式对待彼此
- 接受建设性批评，并以积极的态度回应
- 专注于对社区最有利的事情

## 我们的标准

有助于创造积极环境的行为包括：

- 使用友好和包容的语言
- 尊重不同的观点和经验
- 优雅地接受建设性批评
- 对其他社区成员提供支持和鼓励
- 关注社区的最佳利益而非个人利益
- 展示同理心和友善的态度

不可接受的行为包括：

- 使用性化的语言或意象，以及任何形式的性骚扰
- 发表侮辱性、贬损性或歧视性的评论
- 公开或私下骚扰他人
- 未经明确许可发布他人的私人信息（如地址、电子邮件等）
- 任何其他可能被合理视为不专业的行为

## 责任

项目维护者有责任澄清可接受行为的标准，并对任何被视为不当、威胁、冒犯或有害的行为采取适当和公平的纠正措施。

维护者有权和有责任删除、编辑或拒绝不符合本行为准则的评论、提交、代码、wiki 编辑、问题和其他贡献，并将暂时或永久禁止那些行为不当的贡献者。

## 适用范围

本行为准则适用于所有项目空间，包括但不限于：
- GitHub 仓库（问题、讨论、Pull Request 等）
- 项目相关的聊天室和邮件列表
- 线下活动和会议

当个人代表项目或其社区时，本行为准则也适用于个人的公开场合。代表项目的例子包括使用官方项目电子邮件地址、通过官方社交媒体账号发帖，或在在线或线下活动中担任指定代表。

## 执行

如遇任何违反本行为准则的行为，请通过 [项目维护者联系方式] 向项目维护团队报告。所有投诉将被及时、公平地审查和调查。

维护团队有义务对投诉者的身份保密。具体的执行政策可能会根据情况而有所不同，但维护者将始终努力保持透明和公正。

对于被认定违反本行为准则的行为，维护团队可能会采取包括但不限于以下措施：
- 私下警告
- 公开警告
- 暂时禁止参与
- 永久禁止参与

## 归因

本行为准则基于 [Contributor Covenant](https://www.contributor-covenant.org)，版本 2.0，可在 https://www.contributor-covenant.org/version/2/0/code_of_conduct.html 获取。

社区特定的部分借鉴了其他开源项目的最佳实践，旨在为 SplitMind 社区创建一个积极、包容的环境。
